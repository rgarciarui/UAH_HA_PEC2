Objetivo de la prueba¶


El ejercicio consiste en entregar un Notebook con los siguientes puntos:

  1. Utilizar el API REST de StackEchange (api.stackexchange.com) para obtener una muestra de ms de 1000 preguntas sobre un tag concreto. Por ejemplo, el tag podra ser "python". Convertir esos datos en un DataFrame con al menos los siguientes elementos: identificador de usuario, identificador de pregunta, ttulo, si se ha contestado o no, nmero de veces que se ha visto y fecha de creacin. Sobre ese DataFrame , hacer lo siguiente:
    1. Buscar la pregunta ms antigua no contestada.
    2. Obtener y dibujar la distribucin de las visitas.
  2. Sobre el DataFrame anterior, incluir columnas adicionales obteniendo informacin del API sobre la respuesta ms valorada a cada pregunta, con al menos el identificador de la respuesta, la puntuacin, la fecha en que se cre y el identificador del usuario que respondi.
  3. Sobre el DataFrame anterior, extraer todos los usuarios diferentes que aparecen, y obtener el usuario de mayor y de menor reputacin.

Se va a realizar una consulta mediante un bucle for preguntandole al API con una lista de los ID de las preguntas del dataframe. Con esto se obtiene un nuevo dataframe sobre el que se puede realizar una simplificacin y busqueda.

Los datos que se pretenden tomar para cada pregunta son:

  • Identificador de la respuesta ms valorada (IDAnswer): Se extrae de ['answer_id']

  • Puntuacin (ScoreAnswer): Se extrae de ['score']

  • Fecha en la que se cre (DateAnswer): Se extrae de ['creation_date']

  • Identificador del usuario (IDUserAnswer): Se extrae de ['owner']['user_id']

  • Nombre del usuario (DisplayName): Se extrae de ['owner']['display_name']

  • Reputacion (ReputationIDUserAnswer): Se extrae de ['owner']['reputation']
    • </ul>

    En primer lugar gestionamos el acceso a stackexchange para preguntar sobre las respuestas que se hayan dado a las preguntas, que vamos a extraer en un objeto tipo list. Estos accesos se hacen mediante un bucle de 100 preguntas, ya que si se hace en cantidad superior se bloquea la respuesta de StackAPI.
